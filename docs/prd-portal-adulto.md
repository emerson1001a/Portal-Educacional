# PRD - Portal Adulto

## Objetivo

Definir a experiencia base do adulto no Portal Educacional.

Este PRD organiza regras de UX que devem permear todo o portal adulto, nao
apenas um bloco especifico.

O adulto pode ser pai, mae, professor, orientador ou outro responsavel
autorizado. A tela deve ajudar esse adulto a agir com clareza, sem excesso de
informacao e sem precisar descobrir sozinho onde cada coisa fica.

## Principio central

O portal adulto deve ser organizado em blocos com funcao clara.

Cada bloco precisa responder uma pergunta simples:

```text
Para que esse bloco existe?
```

Se a resposta nao for clara, o bloco deve ser fundido, removido ou redesenhado.

## Estrutura principal

A experiencia adulta deve se dividir em dois grandes grupos:

1. **Registro da Meta e das Missoes**
   - cadastrar ou revisar a intencao pedagogica;
   - criar missoes para a crianca;
   - definir como a crianca acessa o bloco infantil.

2. **Acompanhamento da Crianca**
   - analisar resultados;
   - observar evolucao por area e periodo;
   - abrir evidencias quando necessario;
   - gerar relatorios revisaveis para reunioes.

Essa divisao evita misturar planejamento com analise.

## Regra de blocos recolhidos

Todo bloco adulto deve nascer recolhido, exceto quando houver uma tarefa
imediata que o adulto precisa concluir.

O bloco fechado deve mostrar apenas:

- titulo;
- frase curta explicando a funcao;
- quando fizer sentido, uma pista pequena de prioridade;
- nenhuma lista longa;
- nenhuma evidencia aberta.

O bloco fechado deve ser facil de escanear em poucos segundos.

## Regra de abertura

Quando um bloco ou linha for clicavel como um todo, nao deve haver um segundo
botao redundante de `Abrir` dentro da mesma faixa.

A faixa inteira pode ser a acao principal.

Excecao:

- se o bloco ainda estiver no padrao antigo com botao `Abrir`, ele pode
  permanecer temporariamente;
- na evolucao de UX, o alvo deve ser uma faixa inteira clicavel, com linguagem
  visual clara.

## Regra de recolhimento

Toda area aberta deve ter um botao claro de `Recolher`.

Essa regra vale para:

- blocos principais;
- subareas dentro de um bloco;
- metas abertas;
- missoes abertas;
- evidencias abertas;
- historico aberto;
- relatorios abertos.

O adulto nunca deve ficar preso em uma area longa sem uma saida visual clara.

## Regra de proximidade visual

Itens que resolvem o mesmo problema devem ficar juntos.

Exemplos:

- opcoes de acesso infantil devem ficar no mesmo bloco visual;
- meta e missao podem ficar no mesmo grupo maior, mas com funcoes separadas;
- evidencias devem ficar nos blocos de acompanhamento, nao no bloco de metas;
- relatorio deve ser consequencia da analise, nao um bloco concorrente.

## Regra de acao principal

Cada area aberta deve deixar claro qual e a proxima acao principal.

Evitar:

- muitos botoes com a mesma forca visual;
- botoes pequenos competindo com a faixa principal;
- acao destrutiva sem confirmacao;
- termos tecnicos quando uma frase simples resolve.

Quando houver risco de perda, encerramento ou mudanca importante, o portal deve
confirmar antes.

## Regra de linguagem

A linguagem do portal adulto deve ser:

- clara;
- respeitosa;
- esperancosa;
- pratica;
- sem rotulos;
- sem alarmismo.

O adulto deve sair de cada leitura entendendo:

```text
O que eu devo observar?
O que eu posso fazer agora?
O que ainda e cedo para concluir?
```

## Regra de evidencia

Evidencias sao essenciais, mas nao devem aparecer em todo lugar.

Elas devem ficar principalmente em:

- **Acompanhamento da Crianca**;
- **Acompanhamento por area**;
- **Historico unificado**;
- relatorios revisaveis.

O bloco de metas deve registrar intencao, nao virar area de auditoria.

## Criterios de aceite

O UX do portal adulto estara no caminho correto quando:

- cada bloco fechado tiver funcao clara;
- cada bloco aberto tiver `Recolher`;
- faixas clicaveis nao tiverem botoes extras redundantes;
- planejamento e analise estiverem separados;
- evidencias aparecerem onde ajudam, sem poluir;
- o adulto conseguir entender a tela sem explicacao externa;
- a tela ajudar o adulto a agir, nao apenas ler dados.
